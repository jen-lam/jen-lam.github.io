Background
Habit tracking is the practice of noting down whether a (usually) healthy habit has been maintained. Habits can range from exercising to taking medication to watering plants.

Role: Research, Ideation, Design, Design, Prototyping
Duration: December (1 week)

Problem

Habit tracking is only meaningful if you remember to do it - this is, the habit and the tracking.


Goal

I wanted to create a simple application that not only helps users remember to track a habit but to also help users visualise their progress.

Competitor Analysis
Habit tracking has been around for a long time, whether it be on paper or digitally. I decided to test some existing habit tracking mobile applications along with reading their reviews, to see how they have grown and adapted to industry trends. The following are brief summaries created from reading user reviews:

  • Habit Bull - feature-rich, can't edit an existing habit, UI is a bit overwhelming
  • Loop - clean UI, useful reminders, can't undo mis-taps which leads to inaccurate data, lack of subcategories, can't select certain days
  • Taskaday - simple UI, can only tick of tasks on the current day, only shows progress for the past 2 days
  • Streaks - minimal, no order, can't add notes, buggy
